\section*{The PORT.A Input / Output Connector}

\begin{figure}[H]
    \centering
    \includegraphics[width = .2\linewidth]{m5stack-porta-connector.png}
    \caption{The PORT.A Input/Output Connector}
\end{figure}
\vspace{2mm}
\vspace{5mm}
The connector is used to quickly connect various sensors to the Tab5. It 
enables quick, plug-and-play connectivity for sensors and actuators, often 
powered by the 5V rail. PORT.A can be configured for GPIO (digital I/O), or 
UART serial communication as well as I2C serial communication.  We call this 
Rapid Prototyping because the circuit can be built without building a circuit 
board. 
\vspace{5mm}

This is a 4-pin HY2.0-4P Grove connector.  It's close to a 4 pin JST PH connector.

\vspace{2mm}
\begin{figure}[H]
    \centering
    \includegraphics[width = .8\linewidth]{porta-table.png}
    \caption{PORT.A Pins and their Functions}
\end{figure}
\vspace{2mm}

There are four pins on the PORT.A connector:

\begin{itemize}
    \item Black - GND - This is the common reference terminal against which 
    other outputs are measured.
    \item Red - 5V - This is a 5V reference / power supply from the port, 
    intended to power any sensors or actuators.  Why is it 5V when we're 
    working in a 3.3V ecosystem?  My guess is to let us power and communicate 
    with 5V sensors as well as 3.3V ones.  
    \item Yellow - G53 - This is connected to pin G53 on the microcontroller.  
    It can be turned on and off programmatically. When it's on, it will be 
    +3.3V with respect to ground.  When it's off, it will be 0V.
    \item White - G54 - This is connected to pin G54 on the microcontroller.  
    It can be turned on and off programmatically. When it's on, it will be 
    +3.3V with respect to ground.  When it's off, it will be 0V.
\end{itemize}
\vspace{2mm}
PORT.A has some other special functions which we will cover in later lessons.

\section*{The Dual LED Module}
For any Hello World program, we need an LED to blink.  Unfortunately, 
there's not a blinkable LED built into our Tab5 Dev board (a huge oversight on the part 
of M5Stack).  CrazyUncleBurton didn't let that stop him - he bought some M5Stack 
prototype modules and built his own!
\vspace{2mm}
\begin{figure}[H]
    \centering
    \includegraphics[width = .8\linewidth]{dual-led-module.png}
    \caption{The Dual LED Module Schematic}
\end{figure}
\vspace{2mm}
There are two independent circuits inside the Dual LED Module.  OK, they share 
the ground terminal of the port, but that's it.

The green LED is connected to GPIO pin G53.  A 1K resistor limits current to 
protect the LED and the microcontroller.  When GPIO pin G53 is set to a logic 
high, the ESP32 outputs 3.3V to pin G53 and the green LED lights.  When GPIO 
pin G53 is set to a logic low, the ESP32 outputs 0V to pin G53 and the green 
LED turns off.

The red LED is connected to GPIO G54.  A 1K resistor limits current to protect 
the LED and the microcontroller.  When GPIO Pin G54 is set to a logic high, the 
ESP32 outputs 3.3V to pin G54 and the red LED lights.  When GPIO pin G54 is set 
to a logic low, the ESP32 outputs 0V to pin G53 and the green LED turns off.

Lines 1-5 are a block comment that lets the user know who to yell at when this 
doesn't work.

Line 8 is an "include".  This is a library / module / shared code which adds 
some Arduino-style commands.  Specifically, we want the \textbf{pinMode()} 
command which lets us declare the GPIO pin to be an output, and the \textbf{digitalWrite()} 
command which will turn GPIO pins on and off.

Lines 10 and 11 create global variables of the integer type.  The variables are 
named greenLEDPin, which we set to GPIO 53, and redLEDPin, which we set to GPIO 
Pin 54.  

Lines 15-19 form the setup() function.  This is standard Arduino form, and 
includes all the stuff that only needs to happen once at the time of 
microcontroller turn-on.  The pinMode() commands make the pins referenced by 
the variables \textbf{greenLEDPin} and \textbf{redLEDPin} to be digital outputs.

Digital output pins can output two levels:  
\begin{itemize}
    \item Digital logic level 1, or High, which makes that output pin +3.3V
    \item Digital logic level 0, or Low, which makes that output pin 0V.
\end{itemize}

Lines 22-30 for the loop() function.  This is standard Arduino form, and 
includes the stuff that runs in order from top to bottom.  When it gets to the
end of the list of stuff in the loop function, it loops and runs that stuff 
again. 

\vspace{5mm}

\textbf{Important Point:} Embedded programs are designed to never end 
- they run as long as the device is turned on!

\vspace{5mm}

Lines 24 and 25 sets \textbf{greenLEDPin} to \textbf{High}, which turns the 
green LED on, and sets the \textbf{redLEDPin} to \textbf{Low}, which turns 
the red LED off.

Lines 26 is a delay.  It waits 500 milliseconds (one-half second) before proceding.

Lines 27 and 28 sets \textbf{greenLEDPin} to \textbf{Low}, which turns the 
green LED off, and sets the \textbf{redLEDPin} to \textbf{High}, which turns 
the red LED on.

Lines 26 is a delay.  It waits 500 milliseconds (one-half second) before proceding.
